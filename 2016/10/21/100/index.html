可变参数列表

在JDK1.5起，可以这么做😁

public static void showArrays(Object... x)，不必声明为数组了，因此main方法里可以这么写：
showArrays(new ObjectArrays(),new ObjectArrays());
从foreach语句看出，这个可变参数**依旧是数组，

最后一行说明了：将0个参数传递给可变参数列表是可行的。

在可变参数列表中可以使用任何类型的参数，包括基本类型

不使用参数调用f()时，编译器无法知道具体调用哪一个方法，此时可以在其中一个方法里添加非可变参数来解决

枚举类型

enum Spiciness {
COLD, SO , HOT
}
//上面创建创建了名为Spiciness的枚举类型，它有三个具名值
//按照惯例应该大写
public class TestEnum {
public static void main(String[] args) {
Spiciness felling = Spiciness.COLD;
System.out.println(felling);
System.out.println(felling.ordinal());
}
}
/*
out:
COLD
0
COLD
SO
HOT
*/

创建enum类型时，编译器会自动添加一些有用的特性，例如，创建toString()用来显示某个enum实例的名字，System.out.println(felling);这行代码可以说明。还有ordinal()方法，表示某个enum常量在常量中声明的顺序，以及 static values()方法，按照常量声明顺序，产生这些常量构成的数组
